# Store Intelligence System - Design Document

## 1. Problem Statement

The objective is to process retail store CCTV footage and generate structured visitor events that can be used to derive business intelligence metrics such as:

* Footfall
* Conversion Funnel
* Zone Heatmaps
* Queue Monitoring
* Anomaly Detection
* Store Health Monitoring

The system should support multiple stores and multiple cameras per store.

# 3. Detection Layer

## Model

YOLOv8 Nano

Reason:

* Fast inference
* Lightweight
* Good person detection accuracy
* Easy deployment

Only the person class is detected.

```python
classes=[0]
```

Output:

```json
{
  "track_id": 1,
  "store_id": "STORE_BLR_002",
  "camera_id": "CAM_ENTRY_01",
  "zone_id": "SKINCARE",
  "confidence": 0.91
}
```

---

# 4. Tracking Layer

Tracking IDs generated by YOLO are converted into visitor identifiers.

Example:

```text
track_id = 5

visitor_id = VIS_5
```

Purpose:

* Visitor counting
* Funnel analytics
* Zone analytics
* Session generation

---

# 5. Event Layer

All analytics are derived from a common event schema.

## Event Schema

```json
{
  "event_id": "uuid-v4",
  "store_id": "STORE_BLR_002",
  "camera_id": "CAM_ENTRY_01",
  "visitor_id": "VIS_5",
  "event_type": "ZONE_DWELL",
  "timestamp": "2026-03-03T14:22:10Z",
  "zone_id": "SKINCARE",
  "dwell_ms": 8400,
  "is_staff": false,
  "confidence": 0.91,
  "metadata": {
    "queue_depth": null,
    "sku_zone": "SKINCARE",
    "session_seq": 5
  }
}
```

Benefits:

* Standardized ingestion
* Extensible metadata
* Store-independent analytics

# 6. Analytics Layer

## Metrics

Generated metrics include:

* Unique Visitors
* Revenue
* Conversion Rate
* Queue Depth
* Average Dwell Time

---

## Funnel

Stages:

```text
ENTRY
   ↓
ZONE_VISIT
   ↓
BILLING
   ↓
PURCHASE
```

Used to identify drop-offs.

---

## Heatmap

Heatmap scores are generated using:

* Unique visitors per zone
* Average dwell time
* Normalized visit scores

Example:

```text
SKINCARE   100%
MAKEUP      82%
FRAGRANCE   45%
```

---

## Anomaly Detection

Rule-based anomaly engine.

Current anomalies:

* Low Conversion
* Low Revenue
* Low Footfall
* Queue Congestion
* Low Dwell Time

Output:

```json
{
  "anomaly_type": "LOW_CONVERSION",
  "severity": "HIGH",
  "suggested_action": "Improve merchandising."
}
```
