## Detection

YOLOv8 Nano was selected because:

* Lightweight
* Fast inference
* Good person detection performance
* Easy deployment

## Tracking

YOLO tracking IDs are used to create visitor identifiers.

Format:

```text
VIS_<track_id>
```

## Event Model

Events follow the Purplle schema.

Benefits:

* Easy analytics
* Standardized ingestion
* Extensible metadata

## Heatmap

Zone popularity is computed using:

* Unique visitors per zone
* Average dwell time
* Normalized visit score

## Funnel

Stages:

1. Entry
2. Zone Visit
3. Billing
4. Purchase

## API

FastAPI was chosen because:

* Lightweight
* Automatic Swagger docs
* High performance

## Dashboard

Streamlit was selected because:

* Fast development
* Interactive analytics
* Easy deployment
